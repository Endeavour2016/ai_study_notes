> 创建日期：2026-03-14

## 四大索引概览

```mermaid
flowchart TB
    subgraph VSAG索引家族
```

| 索引 | 数据类型 | 核心结构 | 引入版本 | 适用场景 |
|------|----------|----------|----------|----------|
| **HNSW** | 稠密向量 | 分层小世界图 | 早期 | 通用ANN搜索 |
| **HGraph** | 稠密向量 | 分层图+ODescent | v0.12 | 大规模高性能 |
| **Pyramid** | 稠密向量 | 树形分层图 | v0.14 | 路径过滤搜索 |
| **SINdi** | 稀疏向量 | 分窗口倒排 | - | 稀疏向量搜索 |

### 2. 搜索流程对比

```mermaid
flowchart TB
    subgraph HNSW搜索
        A1[查询向量] --> B1[顶层入口]
        B1 --> C1[贪心下降]
        C1 --> D1[底层搜索]
        D1 --> E1[返回结果]
    end
    
    subgraph HGraph搜索
        A2[查询向量] --> B2[路由图导航]
        B2 --> C2[底层图搜索]

        D2 --> E2[返回结果]
    end
    
    subgraph Pyramid搜索
        A3[查询+路径] --> B3[路径导航]

        D3 --> E3[返回结果]
    end
    
    subgraph SINdi搜索
        A4[稀疏查询] --> B4[词项计算]
        B4 --> C4[遍历窗口]
        C4 --> D4[倒排求交]
        D4 --> E4[返回结果]
    end
```

## 适用场景决策树

```mermaid
flowchart TD
    A[开始选择索引] --> B{数据类型?}
    
    B -->|稀疏向量| C[使用 SINdi]
    B -->|稠密向量| D{数据特点?}
    
    D -->|有路径/分类信息| E[使用 Pyramid]
    D -->|通用数据| F{性能要求?}
    
    F -->|极致性能| G[使用 HGraph]
    F -->|平衡选择| H{是否需要量化?}
    
    H -->|是| I[使用 HGraph]
    H -->|否| J[使用 HNSW]
    
    style C fill:#bfb
    style E fill:#bfb
    style G fill:#bbf
    style I fill:#bbf
    style J fill:#f9f
```

### 场景推荐表

| 场景 | 推荐索引 | 理由 |
|------|----------|------|
| 通用ANN搜索 | HNSW/HGraph | 最成熟、最通用 |
| 大规模数据(>100M) | HGraph | 分层+量化，扩展性好 |
| 内存受限 | HGraph/SINdi | 支持量化压缩 |
| 多租户/分类数据 | Pyramid | 路径天然隔离 |
| 文本Embedding | SINdi | 稀疏向量专用 |
| 推荐系统 | SINdi | 用户-物品矩阵稀疏 |
| 高召回率要求 | HGraph | 重排序机制 |
| 只读静态数据 | HNSW(Static) | 内存紧凑 |

## 总结

### 选择建议

1. **不知道选什么？** → 选 **HNSW**
   - 最成熟、最稳定、社区支持最好

2. **要最好的性能？** → 选 **HGraph**
   - VSAG的旗舰索引，综合性能最佳
   - 支持量化、重排序、属性过滤

3. **有分类/路径信息？** → 选 **Pyramid**
   - 天然支持分层过滤
   - 多租户场景利器

4. **稀疏向量？** → 选 **SINdi**
   - 专为稀疏数据设计
   - 内存和计算都高效

### 发展趋势

```
VSAG索引演进路线:

HNSW ───────┐
            ├──→ HGraph ────→ 未来优化方向
经典算法    │   (v0.12)         - GPU加速
            │                   - 分布式扩展
            └──→ Pyramid        - 更优量化
                (v0.14)

SINdi ───────→ 稀疏向量专用优化
               - 更高效的压缩
               - 更好的过滤支持
```
