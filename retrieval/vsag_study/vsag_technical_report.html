什么是 VSAG？

VSAG 是蚂蚁集团开源的高性能向量检索库，专为海量高维向量的近似最近邻搜索（Approximate Nearest Neighbor Search, ANNS）而设计。

核心问题

  • ▸ 传统数据库难以高效处理高维向量相似度计算
  • ▸ 暴力搜索时间复杂度为 O(N)，无法应对亿级数据
  • ▸ 高维数据的存储与内存成本呈指数级增长

解决方案

  • ▸ 多索引架构：HGraph、Pyramid、SINDI、HNSW、DiskANN 等
  • ▸ 智能量化：FP32/FP16/SQ8/PQ/RaBitQ，压缩率 2x-256x
  • ▸ 灵活存储：内存、磁盘、MMap、异步 IO 多种模式

应用场景类比

向量检索可类比为图书馆检索系统：

  • • 暴力搜索：逐本翻阅所有书籍，精确但效率极低
  • • 索引检索：先通过分类目录（索引）定位，再在目标区域精确查找

核心索引架构

VSAG 提供多种索引结构，针对不同数据特征和应用场景进行优化。以下按照推荐优先级介绍三种核心索引。

HGraph

推荐首选

Hierarchical Graph · 分层图索引

一句话总结：HGraph 是一种分层图索引，结合 HNSW 的多层导航思想和 ODescent 的图构建算法，通过双层量化（粗量化+精量化）和路由图优化，实现高效的近似最近邻搜索。

HGraph vs HNSW：核心差异

图构建算法

HNSW
使用 NSW（Navigable Small World）算法，逐个插入节点，每层贪心搜索最近邻居建立连接
HGraph
使用 ODescent 算法，基于近似 K 近邻迭代优化图结构，构建更高质量的图拓扑

量化与存储

HNSW
主要存储原始向量或简单量化，内存占用较高
HGraph
支持双层量化（基础量化 + 高精度重排序），内存效率更高，支持多种量化类型（SQ8/PQ/RaBitQ）
ODescent
更优的图构建算法
双层量化
粗量化+精量化重排序
路由图
优化的分层导航结构

构建流程

阶段一：训练（Train）

原始数据 ──→ 采样训练数据 ──→ 训练基础量化器 ──→ 训练高精度量化器

void HGraph::Train(const DatasetPtr& base) {
    // 1. 采样训练数据
    DatasetPtr train_data = vsag::sample_train_data(base, ...);
    
    // 2. 训练基础量化编码
    this->basic_flatten_codes_->Train(data_ptr, train_data->GetNumElements());
    
    // 3. 训练高精度编码（用于重排序）
    if (use_reorder_) {
        this->high_precise_codes_->Train(data_ptr, train_data->GetNumElements());
    }
}

阶段二：构建（Build）

HGraph 支持两种图构建方式：

方式一：NSW（导航小世界）
• 逐个插入节点
• 每层贪心搜索最近邻居
• 连接到最近 M 个邻居
• 双向连接并修剪多余边
方式二：ODescent（默认推荐）
• 计算近似 K 近邻
• 迭代优化图结构
• 添加反向边保证连通性
• 修剪低质量边
std::vector<int64_t> HGraph::Build(const DatasetPtr& data) {
    this->Train(data);
    if (graph_type_ == GRAPH_TYPE_VALUE_NSW) {
        return this->Add(data);           // NSW方式：逐个添加
    } else {
        return this->build_by_odescent(data);  // ODescent方式：批量优化
    }
}

贪心搜索算法核心逻辑
DatasetPtr HGraph::KnnSearch(const DatasetPtr& query, int64_t k, ...) {
    // 1. 使用基础编码计算距离
    auto codes = basic_flatten_codes_;
    
    // 2. 贪心搜索找到候选集
    auto candidates = searcher_->Search(query, ef_search);
    
    // 3. 如果需要重排序，使用高精度编码
    if (use_reorder_) {
        candidates = reorder_->Reorder(candidates, query, k);
    }
    
    // 4. 返回 Top-K 结果
    return candidates.TopK(k);
}

存储策略

HGraph 支持多种存储类型，通过参数灵活配置：

memory_io
纯内存存储，访问速度最快，适合数据量可完全放入内存的场景
mmap_io
内存映射文件，操作系统自动管理内存页，适合大内存数据
async_io
异步 IO，重叠计算与 IO 操作，适合磁盘存储场景

Pyramid

分层树形索引

分层树形图索引 · 支持路径过滤

一句话总结：Pyramid 是一种分层树形图索引，通过路径导航和多级子索引结构，将数据按层次组织成金字塔形状，支持基于路径的高效过滤搜索。

2. 节点状态机

每个节点根据数据量自动选择存储策略：

GRAPH
数据量 >= index_min_size，构建图索引进行高效搜索
FLAT
数据量 < index_min_size，使用暴力搜索

索引节点，包含状态、图结构、入口点和子节点映射
base_codes_
向量编码存储，支持多种量化类型
odescent_param_
图构建参数，控制图的质量和密度

核心构建代码
void IndexNode::Build(ODescent& odescent) {
    std::unique_lock lock(mutex_);
    
    // 1. 初始化当前节点
    if (not ids_.empty()) {
        Init();  // 根据数据量决定是 GRAPH 还是 FLAT
    }
    
    // 2. 如果是图节点，构建图
    if (status_ == Status::GRAPH) {
        entry_point_ = ids_[0];
        odescent.SetMaxDegree(static_cast<int32_t>(graph_param_->max_degree_));
        odescent.Build(ids_);
        odescent.SaveGraph(graph_);
    }
    
    // 3. 递归构建子节点
    for (const auto& item : children_) {
        item.second->Build(odescent);
    }
}

SINDI

稀疏向量专用

Sparse Index with Non-zero Dimension Indexing · 稀疏向量索引

一句话总结：SINDI 是一种专为高维稀疏数据设计的索引，通过分窗口存储和倒排列表结构，只存储非零元素，实现高效的稀疏向量相似度搜索。

什么是稀疏向量？

稀疏向量 vs 稠密向量

稠密向量（10000维）：
[0.1, 0.0, 0.0, 0.3, 0.0, 0.5, ...]
  ↓ 存储：10000 × 4字节 = 40KB

稀疏向量表示：
非零元素仅3个：
  ids  = [0, 3, 5]
  vals = [0.1, 0.3, 0.5]
  ↓ 存储：3 × (4+4)字节 ≈ 24字节
  ↓ 压缩比：约 1700:1

典型应用场景

  • ▸ 文本 Embedding：TF-IDF、BM25 等特征，大部分维度为 0
  • ▸ 推荐系统：用户-物品交互矩阵，用户只与少数物品交互
  • ▸ 基因数据：基因表达数据，只在特定基因位点有表达

2. 窗口（Window）机制

SINDI 将数据分窗口存储，每个窗口管理固定数量的向量：

窗口 0（向量 0-9999）     窗口 1（向量 10000-19999）
┌─────────────────┐       ┌─────────────────┐
│ 词项0: [v0, v5] │       │ 词项0: [v10002] │
│ 词项1: [v3, v8] │       │ 词项1: [v10005] │
│ 词项2: [v1]     │       │ 词项2: [v10001] │
└─────────────────┘       └─────────────────┘

优势：
1. 局部性：查询只需访问相关窗口
2. 可扩展：动态添加新窗口
3. 并行：多窗口可并行处理

3. 关键组件

window_term_list_
分窗口存储稀疏向量，每个窗口包含独立的倒排列表
term_id_limit_
最大词项 ID 限制，决定倒排列表数组大小
window_size_
每个窗口的向量数量，影响内存和搜索效率
doc_retain_ratio_
文档保留比例，用于剪枝低频词项

相似度计算原理

稀疏向量的相似度通常使用点积（Dot Product）或余弦相似度：

Query:  {(2, 3.0), (4, 5.0), (6, 2.0)}  ← 只在维度 2,4,6 有值
Doc:    {(0, 1.0), (2, 3.0), (6, 2.0)}  ← 只在维度 0,2,6 有值

相似度计算：
  共同维度：2 和 6
  维度 2：3.0 × 3.0 = 9.0
  维度 6：2.0 × 2.0 = 4.0
  ─────────────────────────
  总相似度：9.0 + 4.0 = 13.0

优势：只需遍历共同非零维度，时间复杂度 O(|Query| + |Doc|)
倒排列表求交优化
通过倒排列表快速找到包含查询词项的文档，避免遍历所有文档。 使用 WAND（Weak AND）或 BMW（Block-Max WAND）算法进行高效求交和剪枝。

核心构建代码
std::vector<int64_t> SINDI::Add(const DatasetPtr& base, AddMode mode) {
    // 1. 动态调整窗口数量
    int64_t final_add_window = align_up(cur_element_count_ + data_num, window_size_) / window_size_;
    while (window_term_list_.size() < final_add_window) {
        window_term_list_.emplace_back(
            std::make_shared<SparseTermDataCell>(doc_retain_ratio_, term_id_limit_, ...)
        );
    }
    
    // 2. 逐个处理向量
    for (uint32_t i = 0; i < data_num; ++i) {
        auto cur_window = cur_element_count_ / window_size_;
        auto inner_id = static_cast<uint16_t>(cur_element_count_ - cur_window * window_size_);
        
        // 3. 插入到对应窗口的倒排列表
        window_term_list_[cur_window]->InsertVector(sparse_vector, inner_id);
        
        // 4. 更新标签映射
        label_table_->Insert(cur_element_count_, ids[i]);
        
        cur_element_count_++;
    }
}

核心搜索代码
DatasetPtr SINDI::search_impl(const SparseTermComputerPtr& computer, ...) {
    MaxHeap heap(allocator);
    Vector<float> dists(window_size_, 0.0, allocator);
    
    // 1. 遍历所有窗口
    for (auto cur = min_window_id; cur <= max_window_id; cur++) {
        auto window_start_id = cur * window_size_;
        auto term_list = this->window_term_list_[cur];
        
        // 2. 计算相似度
        term_list->Query(dists.data(), computer);
        
        // 3. 将结果加入堆
        term_list->InsertHeapByDists<mode>(dists.data(), dists.size(), heap, ...);
    }
    
    // 4. 重排序（如果需要）
    if (use_reorder_) {
        return rerank_by_high_precision(heap);
    }
    
    return collect_results(heap);
}

// 将浮点值量化到8位整数
uint8_t quantize(float val) {
    float normalized = (val - min_val) / diff;  // 归一化到[0,1]
    return static_cast<uint8_t>(normalized * 255);  // 映射到[0,255]
}

// 反量化
float dequantize(uint8_t qval) {
    return min_val + (qval / 255.0f) * diff;
}

量化技术

量化是向量检索中降低存储成本、提升查询效率的关键技术。VSAG 支持多种量化方式，从无损压缩到高压缩比方案。

标量量化（SQ）

将浮点数值映射到较低精度的整数表示

  • • SQ8：8位量化，4x压缩比，精度损失小
  • • SQ4：4位量化，8x压缩比，适合高维数据
  • • SQ8Uniform：均匀分布优化版本

乘积量化（PQ）

将高维向量分割成子空间，每个子空间独立量化

  • • 将 D 维向量分成 M 个子空间
  • • 每个子空间训练码本（通常 256 个中心点）
  • • 存储码本索引而非原始值
  • • 压缩比可达 16x-64x

量化方式对比

量化类型 压缩比 精度损失 计算速度 适用场景
FP32 1x 无 基准 精度敏感场景
